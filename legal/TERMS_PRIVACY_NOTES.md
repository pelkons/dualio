# Dualio Terms And Privacy Notes

This file is not a legal policy. It is a working source-of-truth for generating future Terms of Service, Privacy Policy, AI Processing disclosure, App Store privacy labels, and Google Play Data Safety answers.

## Product Summary

Dualio is an AI-first personal semantic memory inbox. Users save links, screenshots, photos, and text. The app processes saved content to create typed cards, summaries, searchable memory, entities, chunks, and semantic search indexes.

## Current Privacy Position

Dualio is private by account isolation, not zero-knowledge.

Dualio is not end-to-end encrypted while server-side AI processing is enabled. Server-side processing needs readable content to fetch sources, extract text, classify items, generate summaries, create embeddings, and power semantic search.

We should not claim that the project owner or infrastructure administrators technically cannot access user content. The honest position is:

- Users cannot access other users' content.
- Saved content is isolated per account through strict database row-level security.
- We do not sell user content.
- We do not use saved content for advertising.
- We do not manually review saved content except when required for user-requested support, security, abuse prevention, legal compliance, or service operation.
- We do not build internal tools intended for browsing user notes.
- Raw source material should be retained only temporarily and minimized after processing.

## AI Processing

Saved content may be processed by server-side AI systems to:

- Detect item type
- Extract structured fields
- Generate summaries
- Extract entities
- Generate aliases
- Create semantic chunks
- Generate embeddings
- Rerank search results
- Improve search and retrieval quality

The product should disclose that AI processing is a core feature. Users should understand that saved content may be sent to AI subprocessors for processing.

Initial planned AI provider:

- OpenAI API

Important position:

- Do not claim saved content is used to train third-party models unless a provider explicitly does that.
- For OpenAI API, use API data controls according to the current OpenAI policy at implementation time.
- Re-check provider terms before launch.

## Data Storage

Primary database:

- Supabase Postgres

File/object storage:

- Cloudflare R2 for screenshots, photos, thumbnails, and extracted/normalized images

Access model:

- Client uses Supabase anon key only.
- Service role key must never be shipped in the app.
- Edge Functions may use service role for processing pipeline operations.
- R2 access should be through signed URLs generated by trusted backend code.

## Security Controls

Current and planned controls:

- Supabase Row Level Security on user-owned tables
- Per-row `user_id` ownership
- Strict policies for select, insert, update, delete
- No service role key in mobile clients
- HTTPS/TLS in transit
- Encrypted infrastructure/storage where provided by Supabase, Cloudflare, and cloud providers
- Signed URLs for private media
- No raw user content in logs
- No admin content browser
- Least-privilege access for production credentials
- Separate development and production projects before launch
- MFA for production provider accounts
- Rotation plan for service keys and API keys

## Raw Content Retention

Raw content is staging material, not permanent product memory.

Pending/processing items:

- Raw content may be stored while processing is pending or in progress.
- Raw content may be needed for retries.

Ready items:

- Raw content should be minimized or purged after structured data, summaries, chunks, entities, and embeddings are created.
- Store only what is needed to render the card and power search.

Failed items:

- Raw content may be retained temporarily for retry/debug.
- Raw content should expire after a defined window.

Planned schema fields:

- `raw_content_expires_at`
- `raw_content_purged_at`
- `raw_content_retention_reason`

Initial suggested retention:

- Raw HTML/text/OCR staging data: delete after successful processing.
- Failed pending raw data: keep for a short retry/debug window, then purge.
- Original full-resolution images for free accounts: expire after a defined retention window.
- Thumbnails and processed metadata may remain while the item exists.

- Backups may retain data for a limited period according to infrastructure retention.

## Future Private Mode

A future private mode may offer client-side encrypted items.

Tradeoff:

- Private encrypted items would not be processed by server-side AI.
- Semantic search may be unavailable or limited for those items.
- Search may need to run locally or only on unencrypted metadata.

Do not promise this until implemented.

## Legal Drafting Notes

Future legal documents should clearly say:

- Dualio is not end-to-end encrypted.
- AI processing is required for core features.
- User data is isolated by account.
- Raw source material is minimized and purged after processing where possible.
- We do not sell content.
- We do not use content for ads.
- Manual access is limited to support, security, abuse prevention, legal compliance, or service operation.
- Users can request deletion/export when those features are implemented.
